# Changelog

All notable changes to this project are documented here. Format loosely follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/).

## [Unreleased]

### Added
- **`1950s` channel on `/package` ‚Äî postwar-American AI thumbnail prompts.** Second AI-image channel, sibling to Heritage. Same three-panel `HeritagePromptStudio` UI but a different style world: sepia / desaturated warm-tone flanking figures (aged mid-century photograph feel ‚Äî same palette family as Heritage, signalling the era through the visual), modern-realistic-color dramatic center, 1940-1985 era anchor, factory-floor / refinery / steel-mill / suburban-kitchen settings. Voice profile leans on JOBS THAT KILLED, FORGOTTEN TOOLS, decade anchors, and worker-name / city / dollar-amount specificity (avoid-list explicitly bans frontier vocabulary so it doesn't drift into Heritage territory). Center sub-modes expanded from 3 to 6: `auto` (default ‚Äî AI picks the most clickbait-iconic type per script), `tool`, `job` (faceless worker), `food`, `object`, `location`. The prompt route now branches on `auto` and instructs the AI to study the listicle items and pick the single best center type from the channel's allowed set. Two new few-shot examples (`JOBS THAT KILLED FATHERS` in `job` mode + `FORGOTTEN GARAGE TOOLS` in `tool` mode) keep generations on-vibe. `AiThumbnailSpec.styleAnchors` now uses a per-sub-mode `center: Partial<Record<‚Ä¶>>` map instead of three hardcoded fields, so future AI channels just declare the modes they support.
- **Heritage channel on `/package` ‚Äî AI-generated three-panel thumbnail prompts.** New `heritage` channel (listicles about forgotten / lost / dying skills, jobs, foods from the 1800s) selectable in `ChannelPicker`. Because the format (sepia flanking figures + modern-color dramatic center) cannot be produced by a single image-gen call, the package wizard's step 2 swaps `ThumbnailEditor` for a new `HeritagePromptStudio` that emits **three independent prompts** ‚Äî left figure, center subject, right figure ‚Äî with **three variations each** plus a short ALL-CAPS thumbnail title (e.g. "FORGOTTEN FARM TRICKS"). New `POST /api/package/heritage-prompts` endpoint runs GPT-4o with style anchors + few-shot examples baked in (two example thumbnails translated into prompt text) so generations match the channel's vibe; the route enforces sepia-on-flanks and modern-color-on-center, and supports three center sub-modes (`object` / `job` / `food` ‚Äî face NOT shown for `job`). The step indicator relabels "Thumbnail" ‚Üí "Prompts" for AI channels. `POST /api/package/seo` now branches on `imageMode === 'ai'` ‚Äî for AI channels it skips the per-cell `imageKeywords` work and only returns titles + tags + `allKeywords`. `lib/channels.ts` gained an `AiThumbnailSpec` shape, `getAiThumbnailSpec`, and `hasThumbnailSupport` helpers. Heritage state persists in `PackageSession.heritage` so refreshes survive.
- **Kinde auth UX finished ‚Äî login redirect + visible logout.** On production (non-localhost), `app/layout.tsx` now redirects unauthenticated visitors to `/api/auth/login` as a backup to `proxy.ts` (previously the redirect was a TODO comment, so an unauthenticated first-time visitor whose request slipped past the proxy would get a blank server error instead of being asked to log in). The layout also fetches the Kinde user and passes `{ name, email }` to `Header`, which now renders the user's name (falling back to email) plus a **Log out** link pointing to `/api/auth/logout`. Localhost still bypasses auth entirely ‚Äî the header shows no user chip there, matching the existing dev-mode behavior in gridder compose. Gridder's user-capture path was already wired; this change just makes auth legible in the UI to match the sibling `footage-collector` app.

### Changed
- **Cmd/Ctrl+V into `/package` step 2 and `/gridder` now accepts image URLs copied from the web, not just files and bare image-extension URLs.** `useClipboardPaste` previously rejected any pasted URL that didn't end in `.jpg/.png/.gif/.webp/.svg/.bmp/.avif`, so pasting a Google CDN URL, an Unsplash link, or any URL with querystring auth failed silently. The hook now accepts any `http(s)://` URL (the image proxy already validates by fetching) and falls back to extracting `<img src>` from `text/html` clipboard payloads ‚Äî which is what browsers put on the clipboard when you right-click ‚Üí Copy Image. The existing `isInput` guard still prevents URL pastes from triggering when the target is an input/textarea.
- **Image-keyword chips on `/package` step 2 now render above the Images section instead of below the editor.** `KeywordChips` moved inside `ThumbnailEditor` (slotted right before the cells grid) so the user sees the full keyword list before picking which cells to fill. The done-screen `KeywordChips` is unchanged. Hint copy updated ‚Äî it no longer says "the 3 cells above" since the cells are now below the chips.

### Fixed
- **`/package` Analyze no longer fails with "Script text is required" when the script came from a YouTube URL.** `handleAnalyze` calls `setScriptText(transcribed)` and then immediately calls `runSeo()` in the same tick ‚Äî `runSeo`'s `useCallback` closure still held the pre-transcription (empty) `scriptText`, so `/api/package/seo` received `script: ""` and 400'd. `runSeo` now takes an optional `script` opt (same pattern as `scriptType` / `elements`) that `handleAnalyze` passes the fresh transcript through, bypassing the stale closure. The "Regenerate text & keywords" button on step 2 still reads from state, which is correct by then.

### Changed
- **`/api/package/seo` tag generation aligned with `yt-history-script-generator`'s `/metadata` flow.** The tag section of the SEO prompt is now labeled `4. TAGS:` with the exact wording used by `SEO_METADATA_PROMPT` in the reference app ("15-20 relevant YouTube tags for SEO. Mix of broad and specific tags. Include topic keywords, related topics, and long-tail variations."). The server-side post-processing (lowercasing, leading-`#` stripping, dedup, 25-cap) is gone ‚Äî tags are returned to the client exactly as the model emits them, matching the reference's pass-through flow. Only an `Array.isArray` guard remains so the response stays typed.
- **`/package` wizard collapsed from 4 steps to 3: script ‚Üí thumbnail ‚Üí done.** The overlay-generation step (channel customization + ZIP download) is gone for now ‚Äî Analyze now pipes straight into the SEO seed and the thumbnail editor. Removes `handleGenerateOverlays`, the "Download Overlays ZIP" CTA on the done screen, and the `customization` / `zipBase64` fields in the persisted `PackageSession` (legacy sessions with those fields still deserialize; a stored `step === 'overlays'` is collapsed to `'thumbnail'` on restore).
- **`POST /api/package/seo` now returns an `allKeywords` array alongside `imageKeywords`.** `imageKeywords` stays capped to `spec.imageCount` (the top-3 picks that pre-fill the cells). `allKeywords` is an exhaustive 15-25 entry list of searchable subjects found in the script ‚Äî deduped server-side, with every `imageKeyword` hoisted to the front so the cell picks always appear first. `PackageSeoResponse` gained the matching field.

### Added
- **Clickable image-keyword chips on `/package`.** New `KeywordChips` panel renders every `allKeywords` entry as a button; clicking opens Google Images for that keyword in a new tab, and a single "Copy comma-separated" action dumps the whole list to the clipboard. Appears under the thumbnail editor and on the done screen. This replaces the old workaround of going back to `/` to get more keyword options when the 3 pre-seeded cells weren't the right picks.
- **Canva-style hard drop shadow on thumbnail text.** `ThumbnailTextStyle` gained `shadowColor`, `shadowOffset` (px at 1920√ó1080), and `shadowAngle` (degrees; 45 = bottom-right). `drawTextOverlay` now paints an offset fill pass in the shadow color *before* the stroke + main fill, producing the sharp zero-blur drop effect from Canva. The live preview in `ThumbnailEditor` mirrors it with `filter: drop-shadow(‚Ä¶ 0 color)` sized in `em` so it tracks the responsive font size. Garden / listicle ships with `#000000`, offset 12 px, angle 45¬∞.

### Changed
- **Transcription** now uses the Modal-hosted Whisper service (`whisper-transcribe-web.modal.run`) instead of OpenAI Whisper with local ffmpeg/ffprobe chunking. The new service returns word-level timestamps (`{word, start, end}`) which are grouped into segments using a 0.7s pause threshold and an 8s max-segment cap. This produces tighter segment boundaries and more accurate PNG overlay placement.
- **`/api/analyze` minimum gap** between consecutive timestamped elements reduced from **20s ‚Üí 4s** (both the system prompt and the post-processing enforcement in `app/api/analyze/route.ts`). Combined with finer-grained segments from the new transcription service, visual overlays can now land closer to the words that cue them.

### Added
- **`/api/transcribe` now has two modes via a `mode` form field.** `precise` (default) keeps the existing Modal Whisper path with word-level timestamps ‚Äî `/forge` uses this so PNG overlays land on the right word. `fast` re-encodes the audio to opus mono 24 kbps via local `ffmpeg` (keeps it under OpenAI's 25 MB cap, ~10 MB/hour) and calls OpenAI Whisper (`whisper-1` by default, override with `OPENAI_WHISPER_MODEL`); returns segment-level timestamps. Both `/gridder` and `/package` switched to `fast` ‚Äî they only need text + segments, and the per-minute cost is a fraction of Modal.
- **YouTube URL on `fast` mode skips Whisper entirely.** When a `youtubeUrl` is provided to `/api/transcribe?mode=fast`, the route hits the captions-based Railway transcript service (`$YT_TRANSCRIPT_API_URL/transcript`, default `https://youtube-transcript-production-18aa.up.railway.app/api`) and returns the script text directly. No yt-dlp download, no ffmpeg re-encode, no OpenAI cost, no bot-check dance. Falls back to a clear error when the video has no captions ‚Äî the user can then upload the audio file instead. yt-dlp + Whisper still runs for `/forge`-style precise-mode YouTube URLs where word-level timing matters.
- **YouTube URL input on the `/package` script step.** `/api/transcribe` now accepts a `youtubeUrl` form field. The route hits `$YT_TRANSCRIPT_API_URL/transcript` (the captions-based Railway service, default `https://youtube-transcript-production-18aa.up.railway.app/api`) and returns the script text directly. No audio download, no Whisper, no per-request cost, no YouTube bot-check exposure on Railway IPs. Returns a clear error when the video has no captions ‚Äî caller can fall back to uploading the audio file. yt-dlp + python were removed from the Dockerfile and the route entirely; the previous `YT_DLP_PATH` and `YT_DLP_COOKIES_B64` env vars are no longer read.
- **`/package` wizard** ‚Äî new top-level route that produces a complete YouTube package (overlay PNG ZIP + composed thumbnail) per channel. Three steps: script ‚Üí overlays ‚Üí thumbnail. The Garden / listicle channel ships first: 3√ó1 grid, gray gap, black text bar with white top line and yellow bottom line at a 6 px line gap, deterministic image sourcing (Google Images search + paste/upload per cell). Channels live in `lib/channels.ts` so adding new ones is a single config entry.
- **`POST /api/package/seo`** ‚Äî GPT-4o call that, given the script + channel + scriptType, returns **5 CTR-optimized title options** (each tagged with one of 10 psychological principles ‚Äî loss aversion, car crash effect, FOMO, curiosity gap, etc.) with a `primaryText` / `secondaryText` thumbnail-text pair per title, plus the per-cell **image keywords** and **15-20 YouTube SEO tags** generated for **search intent** (3-tier hierarchy: core hook ‚Üí search-intent / how-to phrases that aren't necessarily in the script ‚Üí broad category niches). The top-ranked title is auto-prepended as a clean exact-match tag, then the rest are deduped and lowercased server-side. Each channel has a `voice` profile (audience, signature moves, avoid patterns, example titles) that is injected into the prompt so titles match the channel's tone. Selecting a title in the UI pre-fills the thumbnail's top + bottom lines (still editable). Tags render in a copy-to-clipboard panel (comma-separated or one-per-line). Descriptions are intentionally omitted ‚Äî they come from a separate pipeline.
- **Open Sauce Sans Black** added to `lib/fonts/` (SIL OFL, sourced from `marcologous/Open-Sauce-Fonts`). The Garden thumbnail now renders the **top line in Open Sauce Sans + bottom line in Anton** ‚Äî `ThumbnailTextStyle` was split into `topFont` / `bottomFont` so each channel can mix typefaces. The bar background is now `rgba(0, 0, 0, 0.7)` so the underlying images peek through. Both fonts are also copied to `public/fonts/` and registered with `@font-face` in `globals.css` so the in-browser preview matches the rendered PNG.
- **`POST /api/package/thumbnail-compose`** ‚Äî renders the cells onto a 1920√ó1080 canvas with cover-fit + crop offsets (mirroring Gridder), then draws the channel's text bar overlay at the bottom (auto-fit font size, channel-specified line gap and colors).
- **Script-type classification** in `/api/analyze`. GPT-4o now classifies each script as `listicle | tutorial | explainer | essay | narrative | commentary | other` and returns the result as `scriptType` on `AnalyzeResponse`. Listicle-specific rules (extract every numbered item, `#N ITEM NAME` formatting, `listicle-heading` element type) now apply **only** when `scriptType === "listicle"`. All other types emit `main-title` + `point-of-interest` + `subscribe` only. The server also strips stray `listicle-heading` elements from non-listicle responses as a safety net.
- **Heart drawn as canvas path** for `subscribe` overlays in `lib/canvas.ts`. Replaces the `\u2764` glyph rendered in `serif`, which showed as a tofu box on hosts without an emoji font. No repo font-shipping required.
- `WHISPER_TRANSCRIBE_URL` environment variable ‚Äî override the Modal endpoint used by `lib/transcribe.ts`. Defaults to `https://avoajaugochukwu--whisper-transcribe-web.modal.run`.

### UI
- **Input section** reorganized (`app/components/InputSection.tsx`): audio URL paste is now the primary, highlighted input; audio/script file uploads live inside a collapsed "Other input methods" disclosure; the script-paste textarea is nested inside a second, further-collapsed disclosure.
- Type labels spelled out in element lists: `POI` ‚Üí `Point of Interest`, and the `subscribe` type now renders as `Subscribe` instead of falling through to `POI` (`app/components/ForgeForm.tsx`, `app/components/DownloadArea.tsx`).

### Removed
- Local ffmpeg/ffprobe audio chunking, silence detection, and parallel OpenAI Whisper chunk-transcription. The Modal service (`lib/transcribe.ts`, formerly `lib/audio-chunker.ts`) handles the full pipeline end-to-end.
- `FFMPEG_PATH` / `FFPROBE_PATH` environment variables are no longer read by the transcription path.
