# Changelog

All notable changes to this project are documented here. Format loosely follows [Keep a Changelog](https://keepachangelog.com/en/1.1.0/).

## [Unreleased]

### Changed
- **Transcription** now uses the Modal-hosted Whisper service (`whisper-transcribe-web.modal.run`) instead of OpenAI Whisper with local ffmpeg/ffprobe chunking. The new service returns word-level timestamps (`{word, start, end}`) which are grouped into segments using a 0.7s pause threshold and an 8s max-segment cap. This produces tighter segment boundaries and more accurate PNG overlay placement.
- **`/api/analyze` minimum gap** between consecutive timestamped elements reduced from **20s → 4s** (both the system prompt and the post-processing enforcement in `app/api/analyze/route.ts`). Combined with finer-grained segments from the new transcription service, visual overlays can now land closer to the words that cue them.

### Added
- **`/api/transcribe` now has two modes via a `mode` form field.** `precise` (default) keeps the existing Modal Whisper path with word-level timestamps — `/forge` uses this so PNG overlays land on the right word. `fast` re-encodes the audio to opus mono 24 kbps via local `ffmpeg` (keeps it under OpenAI's 25 MB cap, ~10 MB/hour) and calls OpenAI Whisper (`whisper-1` by default, override with `OPENAI_WHISPER_MODEL`); returns segment-level timestamps. Both `/gridder` and `/package` switched to `fast` — they only need text + segments, and the per-minute cost is a fraction of Modal.
- **YouTube URL input on the `/package` script step.** `/api/transcribe` now accepts a `youtubeUrl` form field; the route shells out to `yt-dlp` (installed in the main `Dockerfile` alongside ffmpeg via `pip3 install yt-dlp`), downloads bestaudio to a temp dir, hands the buffer to the existing Whisper pipeline, and cleans up. No second deploy, no shared bearer token, no paid YouTube transcript API. Override the binary location with `YT_DLP_PATH` if needed; default is `yt-dlp` on `PATH`.
- **`/package` wizard** — new top-level route that produces a complete YouTube package (overlay PNG ZIP + composed thumbnail) per channel. Three steps: script → overlays → thumbnail. The Garden / listicle channel ships first: 3×1 grid, gray gap, black text bar with white top line and yellow bottom line at a 6 px line gap, deterministic image sourcing (Google Images search + paste/upload per cell). Channels live in `lib/channels.ts` so adding new ones is a single config entry.
- **`POST /api/package/seo`** — GPT-4o call that, given the script + channel + scriptType, returns **5 CTR-optimized title options** (each tagged with one of 10 psychological principles — loss aversion, car crash effect, FOMO, curiosity gap, etc.) with a `primaryText` / `secondaryText` thumbnail-text pair per title, plus the per-cell **image keywords** and **15-20 YouTube SEO tags** generated for **search intent** (3-tier hierarchy: core hook → search-intent / how-to phrases that aren't necessarily in the script → broad category niches). The top-ranked title is auto-prepended as a clean exact-match tag, then the rest are deduped and lowercased server-side. Each channel has a `voice` profile (audience, signature moves, avoid patterns, example titles) that is injected into the prompt so titles match the channel's tone. Selecting a title in the UI pre-fills the thumbnail's top + bottom lines (still editable). Tags render in a copy-to-clipboard panel (comma-separated or one-per-line). Descriptions are intentionally omitted — they come from a separate pipeline.
- **Open Sauce Sans Black** added to `lib/fonts/` (SIL OFL, sourced from `marcologous/Open-Sauce-Fonts`). The Garden thumbnail now renders the **top line in Open Sauce Sans + bottom line in Anton** — `ThumbnailTextStyle` was split into `topFont` / `bottomFont` so each channel can mix typefaces. The bar background is now `rgba(0, 0, 0, 0.7)` so the underlying images peek through. Both fonts are also copied to `public/fonts/` and registered with `@font-face` in `globals.css` so the in-browser preview matches the rendered PNG.
- **`POST /api/package/thumbnail-compose`** — renders the cells onto a 1920×1080 canvas with cover-fit + crop offsets (mirroring Gridder), then draws the channel's text bar overlay at the bottom (auto-fit font size, channel-specified line gap and colors).
- **Script-type classification** in `/api/analyze`. GPT-4o now classifies each script as `listicle | tutorial | explainer | essay | narrative | commentary | other` and returns the result as `scriptType` on `AnalyzeResponse`. Listicle-specific rules (extract every numbered item, `#N ITEM NAME` formatting, `listicle-heading` element type) now apply **only** when `scriptType === "listicle"`. All other types emit `main-title` + `point-of-interest` + `subscribe` only. The server also strips stray `listicle-heading` elements from non-listicle responses as a safety net.
- **Heart drawn as canvas path** for `subscribe` overlays in `lib/canvas.ts`. Replaces the `\u2764` glyph rendered in `serif`, which showed as a tofu box on hosts without an emoji font. No repo font-shipping required.
- `WHISPER_TRANSCRIBE_URL` environment variable — override the Modal endpoint used by `lib/transcribe.ts`. Defaults to `https://avoajaugochukwu--whisper-transcribe-web.modal.run`.

### UI
- **Input section** reorganized (`app/components/InputSection.tsx`): audio URL paste is now the primary, highlighted input; audio/script file uploads live inside a collapsed "Other input methods" disclosure; the script-paste textarea is nested inside a second, further-collapsed disclosure.
- Type labels spelled out in element lists: `POI` → `Point of Interest`, and the `subscribe` type now renders as `Subscribe` instead of falling through to `POI` (`app/components/ForgeForm.tsx`, `app/components/DownloadArea.tsx`).

### Removed
- Local ffmpeg/ffprobe audio chunking, silence detection, and parallel OpenAI Whisper chunk-transcription. The Modal service (`lib/transcribe.ts`, formerly `lib/audio-chunker.ts`) handles the full pipeline end-to-end.
- `FFMPEG_PATH` / `FFPROBE_PATH` environment variables are no longer read by the transcription path.
